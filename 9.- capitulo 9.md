## Capítulo 9

### Las fallas del sistema actual

Lo que existe… y lo que no existe

El sistema global funciona.

Pero no está diseñado para resistir.

Está diseñado para optimizar.

Reducir costos.

Acelerar flujos.

Eliminar redundancias.

Ese diseño ha permitido crecimiento.

Pero también ha creado fragilidad.

Las fallas del sistema no son errores.

Son consecuencias.

Resultado directo de cómo fue construido.

El primer problema es la fragmentación.

La información existe.

Pero no está integrada.

Cada actor observa una parte.

Nadie ve el conjunto.

Estados.

Empresas.

Organismos internacionales.

Todos operan con datos parciales.

Eso limita la anticipación.

El sistema reacciona.

Pero no siempre comprende.

El segundo problema es la falta de coordinación.

Los actores no están alineados.

Las decisiones se toman de forma independiente.

A veces contradictoria.

Lo que beneficia a uno puede afectar a otro.

Y esa falta de coherencia reduce la efectividad.

El sistema no actúa como uno.

Actúa como muchos.

El tercer problema es la reacción tardía.

Las respuestas se activan después del evento.

Cuando el impacto ya comenzó.

Cuando el efecto dominó ya está en curso.

Ese retraso amplifica el daño.

El sistema llega.

Pero llega tarde.

El cuarto problema es la ausencia de redundancia equivalente.

Existen alternativas.

Pero no equivalentes.

No replican capacidad.

No replican eficiencia.

Eso limita la capacidad de absorción.

El sistema puede desviarse.

Pero no sin costo.

El quinto problema es la dependencia de confianza.

El sistema no se sostiene solo con infraestructura.

Se sostiene con expectativas.

Con estabilidad percibida.

Cuando esa confianza se erosiona, el sistema cambia.

Las decisiones se vuelven conservadoras.

Los costos aumentan.

El flujo se altera.

El sexto problema es la falta de visión sistémica.

Las soluciones se diseñan de forma local.

Para puntos específicos.

Pero el riesgo es global.

Y esa desconexión genera brechas.

Se protege el punto.

Pero no la red.

Estos problemas no operan de forma aislada.

Se refuerzan.

Se amplifican.

Y generan una vulnerabilidad estructural.

El sistema no falla por una causa única.

Falla por acumulación.

Desde una perspectiva estratégica, el riesgo no es solo la disrupción.

Es la incapacidad de respuesta.

La falta de anticipación.

La fragmentación.

Ese es el verdadero problema.

Desde una perspectiva adversaria, estas fallas son oportunidades.

No es necesario crear vulnerabilidad.

Ya existe.

Solo hay que identificarla.

Y activarla.

Las capacidades actuales permiten operar el sistema.

Pero no siempre protegerlo.

Ese es el límite.

Una doctrina efectiva requiere cerrar estas brechas.

Primero, integración de información.

Visión completa.

Segundo, coordinación entre actores.

Acción conjunta.

Tercero, anticipación.

Actuar antes del evento.

Cuarto, desarrollo de redundancias funcionales.

No equivalentes.

Pero suficientes.

Quinto, fortalecimiento de confianza.

Reducir incertidumbre.

Estas medidas no eliminan la vulnerabilidad.

Pero reducen su impacto.

El sistema global no es frágil por accidente.

Es frágil por diseño.

Y entender ese diseño es el primer paso para corregirlo.

Porque mientras las fallas permanezcan, el riesgo no desaparecerá.

Solo cambiará de forma.
